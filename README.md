# Example of pl/sql project structure

```
<project-name>
<project-name>/db
<project-name>/db/<SID>
<project-name>/db/<SID>/<scheme>
<project-name>/db/<SID>/<scheme>/dba
<project-name>/db/<SID>/<scheme>/ddl
<project-name>/db/<SID>/<scheme>/dml
<project-name>/db/<SID>/<scheme>/plsql
<project-name>/db/<SID>/<scheme>/plsql/contexts
<project-name>/db/<SID>/<scheme>/plsql/directories
<project-name>/db/<SID>/<scheme>/plsql/functions
<project-name>/db/<SID>/<scheme>/plsql/java
<project-name>/db/<SID>/<scheme>/plsql/jobs
<project-name>/db/<SID>/<scheme>/plsql/mviews
<project-name>/db/<SID>/<scheme>/plsql/packages
<project-name>/db/<SID>/<scheme>/plsql/procedures
<project-name>/db/<SID>/<scheme>/plsql/queues
<project-name>/db/<SID>/<scheme>/plsql/triggers
<project-name>/db/<SID>/<scheme>/plsql/types
<project-name>/db/<SID>/<scheme>/plsql/views
<project-name>/db/<SID>/<scheme>/tests
```
